5️⃣ Session & Token Refresh

Dual-Layer Token Storage

localStorage — Full Supabase session (access + refresh token). Used by Supabase JS SDK.
Cookie — access_token mirror, 1hr expiry. Used by Next.js edge middleware.

Auto-Refresh Flow

authedFetch() sends Bearer token
Backend returns 401 → triggers refresh
supabase.auth.refreshSession() → new token
Retries request (once only)
If refresh fails → returns 401 (no forced logout)

Known Issues & Observations

Issues discovered during auth flow testing on 2026-03-23.

⚠ Signup does not redirect after success

After successful signup, user sees "Account created!" message but stays on /auth/signup. They must manually navigate to login or wait for verification email. Consider redirecting to a "check your email" page.

UX auth/signup

⚠ "Remember me" checkbox is UI-only

The checkbox exists on both login and signup pages but has no backend persistence. Session always expires in 1 hour regardless. Either implement extended sessions or remove the checkbox.

UX auth/login auth/signup

⚠ Password reset route unverified

"Forgot Password?" links to /admin-pages/auth_pages/password_reset. This route needs verification that it exists and works correctly end-to-end.

Verify auth/login

🔴 Onboarding is not enforced

Dashboard renders even if onboarding_complete = false. There's only a profile completeness nudge (< 70% threshold). Consider forcing onboarding redirect for new users.

Logic Gap dashboard onboarding

ℹ Logout API is a no-op

POST /api/auth/logout returns { success: true } without doing anything server-side. All cleanup happens client-side (clear state, cookie, localStorage). Fine for now but limits server-side session revocation.

Low Priority api/auth

ℹ Profile completeness threshold

HeaderMenu shows "Complete profile" button when getProfileCompleteness() returns < 70%. This nudges users to /account/profile. The threshold and scoring logic are in UserProfileContext.

Info header
