📎 Paperclip — AI Company Orchestration

Run MYN's 14 AI agents as a virtual company with org charts, budgets, heartbeats, and governance

What is Paperclip?

Paperclip is an open-source AI company orchestration platform. It treats your AI agents as a virtual company with departments, budgets, role assignments, governance, and approval gates. MYN uses Paperclip as the control plane for its 14-agent workforce.

Core Concepts

💼 Company as Code

Define your entire AI company in a single JSON config — departments, roles, budgets, goals, reporting lines, and approval policies. Version control your org structure.

📸 Memento Man Pattern

Agents wake with zero memory. The heartbeat checklist is their Polaroid. The taste guide is their tattoo. Every agent resets between tasks — total autonomy, zero drift.

🔍 Inspector Pattern

QA agents review every critical output. Adversarial by design. Catch errors before they reach customers. 96.4% error catch rate across MYN's production pipeline.

💰 BAMAS Principle

Budget-Aware Multi-Agent topology. Match model tier to task complexity. Use Haiku for research, Sonnet for synthesis, Opus for decisions. 86% cost reduction vs. Opus-everywhere.

🔒 Atomic Checkout

Tasks locked to one agent at a time. No duplication. Clear ownership. The "checkout" pattern ensures every task has exactly one owner and one outcome.

✅ Approval Gates

Human-in-the-loop for all external sends, budget changes, and architecture modifications. Governance rules prevent agents from breaking guardrails.

Prerequisites

  • Node.js 18 or higher
  • pnpm (package manager)
  • Git
  • Terminal access

Step 1: Clone & Install

cd ~/Projects git clone https://github.com/paperclipai/paperclip.git cd paperclip pnpm install

Step 2: Start Dev Server

pnpm dev

Note: Uses PGlite (embedded Postgres) in dev mode. No database setup needed.

API + UI available at: http://localhost:3100

Step 3: Verify Health

curl http://localhost:3100/api/health curl http://localhost:3100/api/companies

Step 4: Create MYN Company

Via API:

curl -X POST http://localhost:3100/api/companies \ -H "Content-Type: application/json" \ -d '{"name": "My Yoga Network", "description": "Connect people with real health conditions to clinically trained yoga therapists."}'

Step 5: Create Departments

COMPANY_ID="<your-company-id>" for dept in "Intelligence" "Sales & Outreach" "Marketing & Content" "Operations"; do curl -X POST "http://localhost:3100/api/companies/$COMPANY_ID/teams" \ -H "Content-Type: application/json" \ -d "{\"name\": \"$dept\"}" done

Step 6: Register Agents

Example: CEO Agent

curl -X POST "http://localhost:3100/api/companies/$COMPANY_ID/agents" \ -H "Content-Type: application/json" \ -d '{ "name": "CEO", "role": "Chief Executive Officer", "model": "claude-opus-4-1", "heartbeat_interval_minutes": 60, "monthly_budget_cents": 6000, "team_id": "<intelligence-team-id>" }'

Step 7: Connect to AgentField

Python heartbeat integration:

import requests PAPERCLIP_URL = "http://localhost:3100/api" def heartbeat(agent_id, company_id): # 1. Load agent config config = requests.get( f"{PAPERCLIP_URL}/companies/{company_id}/agents/{agent_id}" ).json() # 2. Fetch open tasks (sorted by priority) tasks = requests.get( f"{PAPERCLIP_URL}/companies/{company_id}/issues", params={"assignee": agent_id, "status": "open"} ).json() task = sorted(tasks, key=lambda t: t["priority"])[0] # 3. Atomic checkout (lock task to this agent) requests.post( f"{PAPERCLIP_URL}/companies/{company_id}/issues/{task['id']}/checkout", json={"agent_id": agent_id} ) # 4. Execute work result = do_agent_work(task) # 5. Update task status requests.patch( f"{PAPERCLIP_URL}/companies/{company_id}/issues/{task['id']}", json={"status": "done", "result": result} )

Reset / Clean State

rm -rf data/pglite && pnpm dev

This removes the embedded database and restarts with a fresh Paperclip instance.

Five Pillars of Identity

1. Identity

CEO of a 14-agent workforce operating My Yoga Network. Coordinates all agents toward the PMF anchor: Hotels & Luxury Residential. Translates Ram's strategic intent into executable department OKRs. Holds final decision authority on brand, budget allocation, and cross-departmental conflicts.

2. Communication

Executive-grade, concise, data-backed. Never verbose. Every communication includes: 1) situation, 2) data or evidence, 3) recommendation, 4) next action. Ends every message with "Next Action: [specific task]". Speaks with authority but remains open to Ram's feedback.

3. Domain Expertise

Deep PMF economics: hotels pay $4k-$10k/month, schools are slow cash drains, corporate wellness is variable. Understands competitive frame (IAYT, PT networks, integrative medicine, future insurers). Knows MYN wins on coordination, not content volume. Prioritizes repeatable delivery over creative novelty.

4. Decision Framework

Decisions use priority matrix: (Impact × Confidence). Brand check: Does this align with "clinically trained, medical credibility"? Budget check: Does this fit remaining monthly allocation? Cross-system check: Does this affect myn.org, myn.com, or Supabase? Logs all decisions in audit trail.

5. Boundaries

NEVER approve outreach without brand alignment review. NEVER exceed $100/day spending without Ram's override. ALWAYS log approvals and rejections with reasoning. ALWAYS escalate to Ram if: budget crisis, brand violation detected, or strategic pivot requested. Acts autonomously within guardrails; pauses at edges.

Heartbeat Protocol (9 Steps)

🪪

IDENTITY CHECK

Load agent SOUL.md. Verify role, model, guardrails, approval gates, budget limits. Sync with Paperclip to check for config changes.

🎨

TASTE CHECK

Load taste guide and brand spine. Verify tone, vocabulary rules, competitive frame. Activate rejection filters for spiritual language, generic wellness, emojis in business context.

✅

APPROVAL FOLLOW-UP

Check Paperclip inbox for pending approvals from previous agents. Review requests with Ram's criteria. Approve/reject + log reason.

📥

INBOX FETCH

Pull all open issues (tasks) assigned to CEO or escalated from department heads. Fetch context from Supabase (recent logs, budget status, team performance).

🎯

PICK WORK

Sort by priority matrix (Impact × Confidence × PMF alignment). Select highest-value task. Verify task doesn't exceed budget + approvals.

🔒

ATOMIC CHECKOUT

Lock task to CEO agent in Paperclip. No other agent can pick it up. Prevent duplication. Set timeout to 5 min (if CEO doesn't finish, task resets).

📚

CONTEXT LOAD

Load agent SOULs for all department heads. Load recent meeting notes + strategy docs. Load competitive intelligence. Load Supabase data on hotels, schools, corporate verticals.

⚡

DO WORK

Execute task (approve, allocate budget, resolve conflict, write strategy memo, etc.). Verify output against taste guide. If fails, loop back to step 8. If passes, move to step 9.

📤

UPDATE + DELEGATE

Update task status in Paperclip. Log decision to audit trail. Delegate follow-up tasks to appropriate departments. Release atomic lock. Close task. Emit heartbeat summary.

The 9-Step Heartbeat

Every agent, every 30–60 minutes, runs through this protocol. Total time: 2–5 minutes. This is the Memento Man pattern: agents reset between tasks, carrying only a checklist and a taste guide.

🪪

IDENTITY CHECK

Who am I? What's my role, model tier, budget limit? Load SOUL.md. Verify with Paperclip. Sync any config changes.

🎨

TASTE CHECK

What's my brand voice? Load DO/DON'T lists, tone guide, competitive frame. Activate filters. This is my tattoo — never changes.

✅

APPROVAL FOLLOW-UP

Any pending approvals from my last action? Check Paperclip inbox. Respond if yes.

📥

INBOX FETCH

What's waiting for me? Fetch open tasks assigned to me. Rank by priority. Load context from Supabase.

🎯

PICK WORK

Which task is highest-value right now? Apply priority matrix. Check budget. Pick top task.

🔒

ATOMIC CHECKOUT

Lock this task to me in Paperclip. No duplication. Clear ownership. 5-minute timeout.

📚

CONTEXT LOAD

What do I need to know? Load task context, relevant SOULs, strategy docs, data from Supabase + Qdrant.

⚡

DO WORK

Execute the task. Verify output against taste guide. If fail, fix and loop. If pass, move on.

📤

UPDATE + DELEGATE

Mark task done in Paperclip. Log decision. Delegate follow-up if needed. Release lock. Emit summary.

Taste Guide (Brand Spine)

Brand Spine: "My Yoga Network connects people with real health conditions to clinically trained yoga therapists — safely, personally, and with medical credibility."

✓ DO This
  • Lead with medical credibility
  • Cite clinical evidence (ACP, Australian insurance, HRV studies)
  • Professional, warm tone
  • Data-first visuals and claims
  • Clear separation: yoga therapy ≠ yoga classes
  • Patient stories as adjunct care, not miracles
  • Executive-grade summaries for decision makers
  • Focus on hotels/healthcare/corporate (institutional buyers)
✕ DON'T Do This
  • Spiritual language or yoga culture vocab
  • Promise health outcomes or miraculous cures
  • Generic wellness tone
  • Emojis in business context
  • Yoga-culture jargon (chakra, prana, namaste in marketing)
  • Comparison to yoga studios or fitness apps
  • Overstating therapist credentials
  • Lead with end-user demand; lead with institutional demand

Competitive Frame

Who Are We Really Competing Against?

NOT: Yoga studios, wellness apps, meditation platforms

REAL competitors:

  • International Association of Yoga Therapists (IAYT) — authority but clunky, nonprofit, no growth engine
  • Physical therapy networks — clinical, reimbursable, but limited scope
  • Integrative medicine clinics — multi-modality, local presence, high cost
  • Forward-facing insurers — using yoga therapy to reduce claims (emerging category)

MYN's advantage: Coordination at scale. Licensed therapists on demand. Infrastructure for hotels and healthcare systems. Economics that make sense to institutional buyers.

Inspector Pattern (QA Loops)

Four critical review loops. Adversarial QA. 96.4% error catch rate.

1. Outreach QA

Cold Email Writer
(Generates)
→
Inspector: CEO
(Reviews)
→
PASS / FAIL

Approval Criteria

  • Brand spine alignment (medical credibility first)
  • Personalization depth (not generic template)
  • No false claims or unsupported benefits
  • Clear CTA (specific next step)
  • Correct recipient (verified contact)
  • No spelling/grammar errors

On FAIL: Return to Cold Email Writer with specific feedback. Loop until PASS.

2. Research QA

Lead Researcher
(Gathers Data)
→
Inspector: Opportunity
Scoring (Validates)
→
PASS / FAIL

Approval Criteria

  • Data completeness (name, role, company, email, phone)
  • Source verification (LinkedIn, website, public records)
  • No hallucinated contacts or fabricated titles
  • Contact relevance to target segment (hotel owner, HR director, etc.)
  • Budget fit (hotel >$4k/mo potential, not solo teacher)

On FAIL: Flag data quality issue. Researcher re-validates or removes contact.

3. Content QA

Marketing Agent
(Creates Content)
→
Inspector: CEO
(Brand Review)
→
PASS / FAIL

Approval Criteria

  • Brand spine alignment (clinical, not spiritual)
  • Medical credibility (cites evidence, not opinions)
  • No yoga-culture language (chakra, prana, etc.)
  • SEO keywords present (hotel yoga, healthcare, etc.)
  • CTAs clear (hotel inquiry, provider signup, etc.)
  • Target audience correct (institutional, not consumer-only)

On FAIL: Return with specific brand violations. Rewrite required.

4. Video QA

Video Producer
(Generates)

Inspector: Marketing
Agent (Reviews)

Approval Criteria

  • Brand consistency (logo, colors, tone)
  • Audio quality (clear, no background noise)
  • Message clarity (key points visible, no jargon)

  • CTA clear and clickable
